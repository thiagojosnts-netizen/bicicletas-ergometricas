## 1. POWER KEYWORDS

### 1.1 Clusters Informacionais

**CLUSTER 1 — Autoridade e Controle Semântico**
Uso: sinalizar domínio do assunto.
Guia completo, Guia definitivo, Entenda de vez, O que realmente importa, Tudo o que você precisa saber, Explicação completa, Análise aprofundada, Visão especializada, Análise crítica, Abordagem técnica

**CLUSTER 2 — Educação, Aprendizado e Processo**
Uso: IA educacional, correção de redação.
Como aprender, Desenvolvimento real, Evolução prática, Aprendizado consciente, Formação sólida, Escrita orientada, Prática estruturada, Método de aprendizado, Construção gradual, Desenvolvimento contínuo

**CLUSTER 3 — Ética, Segurança e Confiança**
Uso: IA, plágio, avaliações.
Uso ético, Uso responsável, Uso consciente, Dentro das regras, Sem penalizações, Conformidade acadêmica, Prática segura, Critérios acadêmicos, Avaliação justa, Integridade acadêmica

**CLUSTER 4 — Diferenciação e Posicionamento**
Uso: evitar conteúdo genérico.
Diferença entre, O que muda na prática, Quando vale a pena, Onde está o erro, O que poucos explicam, O que ninguém te conta, Como funciona de verdade, O que faz diferença, Abordagem correta, Uso adequado

**CLUSTER 5 — Intenção Investigativa e Comparativa**
Uso: H1, H2 e snippets.
Vale a pena, Quando usar, Quando não usar, Prós e contras, Vantagens e limitações, Benefícios reais, Riscos envolvidos, Limites do uso, Comparação prática, Análise objetiva

**CLUSTER 6 — Método, Critério e Estrutura**
Uso: SEO semântico e GEO.
Passo a passo, Critérios de avaliação, Método correto, Processo estruturado, Avaliação criterial, Análise por competências, Estrutura ideal, Modelo recomendado, Abordagem orientada, Padrões acadêmicos

**CLUSTER 7 — Resultado e Transformação**
Uso: com cuidado, sem promessas irreais.
Resultados consistentes, Evolução mensurável, Melhoria contínua, Escrita mais clara, Argumentação sólida, Desenvolvimento autoral, Aprimoramento textual, Progresso real, Ganho de clareza, Escrita mais segura

**CLUSTER 8 — Problemas, Erros e Alertas**
Uso: capturar atenção sem alarmismo.
Erros comuns, Principais falhas, O que evitar, Onde muitos erram, Riscos reais, Problemas frequentes, Limitações práticas, Uso incorreto, Dependência excessiva, Falhas de aprendizado

**CLUSTER 9 — IA, Tecnologia e Modernidade**
Uso: específicas e sóbrias, sem hype.
IA aplicada à educação, Inteligência artificial na escrita, IA como ferramenta de apoio, Feedback automatizado, Avaliação com IA, Correção assistida por IA, Tecnologia educacional, IA no aprendizado, Uso pedagógico da IA, IA orientada por critérios

**CLUSTER 10 — Clareza, Verdade e Transparência**
Uso: GEO e LLMs.
De forma clara, Sem atalhos, Sem promessas irreais, O que é e o que não é, Limites e possibilidades, Na prática, Em contexto real, Com exemplos, De maneira objetiva, Explicado passo a passo

### 1.2 Keywords Comerciais

**Decisão e Compra:** Vale a pena, Melhor opção, Melhor escolha, Solução ideal, Alternativa confiável, Opção recomendada, Escolha certa, Melhor custo benefício, Opção mais segura, Solução eficiente

**Comparação e Avaliação:** Comparativo, Comparação completa, Análise comparativa, Diferença entre, Comparado a, Avaliação completa, Teste prático, Análise detalhada, Avaliação real, Comparação honesta

**Valor e Retorno:** Custo benefício, Investimento inteligente, Retorno esperado, Ganho real, Economia de tempo, Redução de esforço, Eficiência comprovada, Valor agregado, Impacto prático, Benefício direto

**Confiança e Prova:** Funciona mesmo, Resultados reais, Uso comprovado, Solução confiável, Método validado, Experiência prática, Casos reais, Testado na prática, Avaliado por usuários, Comprovação prática

**Adoção e Uso:** Como escolher, Quando usar, Para quem é indicado, Para quem não é indicado, Como aplicar, Implementação prática, Uso recomendado, Adotar a solução, Escolher corretamente, Tomar decisão

### 1.3 Regras de Uso

- Informacionais: usar em H1, H2, meta descriptions, primeiro parágrafo, FAQ
- Comerciais: usar em landing pages, comparações, CTAs, seções de decisão
- Distribuir naturalmente — nunca acumular múltiplas power keywords no mesmo parágrafo
- Combinar com entidades do tópico para reforçar relevância semântica

## 2. GLOSSÁRIO TÉCNICO UNIFICADO

| Termo | Definição |
|-------|-----------|
| **BERT** | Bidirectional Encoder Representations from Transformers. Modelo de processamento semântico bidirecional do Google (2018). Base do entendimento de queries. |
| **Bi-word** | Par de palavras adjacentes, usado como unidade de indexação. |
| **Centroid Vector** | Vetor que representa o centro geométrico de um cluster de documentos. |
| **CiC Prompting** | Corpus-in-Context. Inserir corpus completo na janela de contexto do LLM em vez de usar RAG. |
| **COINC Score** | Métrica de coincidência semântica: POS_SIM - NEG_SIM (phrase) ou POS_SIM (freetext). |
| **Cosine Similarity** | Similaridade entre vetores baseada no ângulo: cos(α) = (D1·D2) / (‖D1‖×‖D2‖). |
| **E.E.A.T.S.** | Entity-Encoded Article Topical Structure. Framework de 7 camadas para análise semântica de conteúdo. |
| **EAV** | Entity-Attribute-Value. Modelo de organização de conhecimento: Entidade + Atributo + Valor. |
| **Embedding** | Representação vetorial de palavras, frases ou entidades num espaço multidimensional. |
| **Entity Lock-in** | Camada 1 do E.E.A.T.S.: fixação inequívoca da entidade-alvo nos primeiros 100 palavras. |
| **Essential Entity Set** | Camada 2 do E.E.A.T.S.: subentidades obrigatórias que o espaço vetorial espera. |
| **FREQ** | Frequência do significado no corpus. Termos raros (FREQ baixo) têm maior peso informacional. |
| **FSW** | Frequency-balanced Semantic Weight: FSW = SWC / (1 + log₂(FREQ)). |
| **Generalized Entity** | Unidade semântica de uma ou mais palavras stemizadas, usada na indexação. |
| **GEO** | Generative Engine Optimization. Otimização para ser citado/recuperado por motores gerativos. |
| **Holonymy** | Relação "todo que contém parte": body → arm. |
| **Hypernymy** | Relação "IS-A parent" (categoria superior): dog → animal. |
| **Hyponymy** | Relação "IS-A child" (subtipo): animal → dog. |
| **Information Gain** | Quanto um conteúdo reduz a incerteza do sistema sobre o tópico. |
| **ISC** | Category Index. Importância proporcional à categoria semântica. |
| **LF** | Lexical Function. Ferramenta formal do MTT para representar relações entre unidades lexicais. |
| **LFR** | Lexical Function Rules (Expansion, Contraction, Transformation). |
| **LOFT** | Long Context Frontiers. Benchmark do Google DeepMind para avaliar LLMs com contexto longo. |
| **LSA** | Latent Semantic Analysis. Superado por embeddings neurais. |
| **LSC** | Lemma + Semantic Category. Representação semântica unitária de uma palavra. |
| **LSCS** | Sequência de LSCs representando significado completo de uma expressão. |
| **LSE** | Latent Semantic Entities. Embeddings de entidades detectados por modelos neurais em grafos semânticos. |
| **LSI** | Latent Semantic Indexing. Técnica de representação vetorial (1990s). "LSI keywords" no mercado = LSE. |
| **Meronymy** | Relação "parte de um todo": arm → body. |
| **MTT** | Meaning-Text Theory. Teoria linguística onde significado é definido por combinações lexicais. |
| **MUM** | Multitask Unified Model. 1000x BERT, 75 idiomas, multimodal (Google, 2021). |
| **NEG_SIM** | Similaridade negativa: penalização por conteúdo não correspondido. |
| **Non-intersecting Cluster** | Cluster onde nenhum documento pertence a múltiplos clusters simultaneamente. |
| **POS_SIM** | Similaridade positiva: soma de (FSW × SAF) para todos os matches. |
| **PPS** | Partial Positive Similarity: FSW₁ × SAF para um par de LSCs. |
| **RAG** | Retrieval-Augmented Generation. Recuperação de contexto + geração via LLM. |
| **RankBrain** | Sistema de ML do Google para interpretar queries (anterior ao BERT). |
| **REL Factor** | Fator de confiabilidade do match baseado na posição (título > H2 > body). |
| **Rich Query** | Query do Usuário + Vetor de Contexto Local + Vetor de Perfil do Usuário. |
| **SAF** | Semantic Approximation Factor: 1.00 (exato), 0.99 (Syn0), 0.75 (Syn1), 0.50 (Syn2), 0.60 (Ao). |
| **Seed Exemplar** | Documento representativo de um cluster semântico. |
| **Semfield** | Campo semântico amplo que agrupa múltiplos synsets (ex: "agricultura"). |
| **Semantic Distance** | Distância conceitual entre uma seção/página e a entidade central do site. |
| **SGE** | Search Generative Experience. Sistema do Google para respostas generativas na SERP. |
| **Stemming** | Redução de palavras à raiz morfológica (ex: "correndo" → "corr"). |
| **Stop-word** | Palavra muito comum sem contribuição para relevância semântica. |
| **SWC** | Semantic Weight Coefficient: ISC normalizado (soma = 1). |
| **Synset** | Conjunto de sinônimos representando um conceito único (WordNet), com ID numérico. |
| **T5** | Text-to-Text Transfer Transformer. Framework base do MUM. |
| **Topical Authority** | Autoridade semântica construída por cobertura completa, consistente e conectada de um tópico. |
| **Topical Map** | Ontologia editorial hierárquica: entidade central → subentidades → atributos → processos. |
| **Tríade SERP** | Document Ranking → Passage Ranking → Passage Generation. Três camadas interdependentes. |
